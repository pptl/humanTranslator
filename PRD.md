# PRD：人類翻譯機 / Human Translator

**版本**：v1.0  
**日期**：2026-04-14  
**狀態**：草稿

---

## 1. Why — 產品存在的理由

### 核心問題

學習英文的人往往缺乏「主動輸出」的機會，學到的詞彙也與日常生活脫節。現有工具要麼只做翻譯、要麼只改文法，沒有一個工具能讓人**用自己的真實生活句子練習翻譯，並獲得即時、有深度的 AI 回饋，同時追蹤弱點持續復習**。

### 使用者痛點

- 學了英文但缺乏「主動輸出」的機會
- 學到的詞彙與日常生活脫節
- 缺乏即時回饋來知道自己哪裡錯了
- 沒有系統追蹤弱點、強化記憶

### 最終目標

讓使用者能以英文流暢書寫日常對話。

---

### P0｜MVP 核心功能

#### 3.1 懸浮視窗

- 應用啟動後，畫面上始終有一個小型懸浮視窗**置頂顯示**
- 點擊懸浮視窗 → 展開完整輸入介面
- 使用完畢 → 縮回懸浮視窗
- 支援 Windows / macOS

#### 3.2 翻譯批改

**使用者輸入：**
1. 原文句子（中文）
2. 自己的英文翻譯
3. 情景選擇（見 3.3）

**AI 回饋格式：**

```
✅ 判斷：意思正確，但表達不夠自然（7/10）

📝 你的版本問題：
為什麼你的翻譯有問題的解釋

💡 更好的說法：
一個最佳版本

🔄 不同情景的版本：
・正式場合：「...」
・朋友對話：「...」
・書面寫作：「...」

📚 相關語法小提示：
一兩句，點到為止
```

#### 3.3 情景系統

- 下拉選單快速選擇已儲存的情景
- 可臨時輸入新情景
- 新情景下方有 checkbox「儲存此情景」（**預設勾選**）
- 情景資料儲存於本機

#### 3.4 API Key 設定

- 使用者可自行輸入 LLM API Key
- MVP 階段支援：Claude（Anthropic）、OpenAI
- Key 儲存於本機，不上傳

---

### P1｜重要功能（MVP 後第一批）

#### 3.5 復習系統

- 每次批改結果自動儲存至本機
- 間隔重複演算法（類 Anki）定期重考錯誤句子
- 復習時顯示：
  - 原始句子
  - 當初的錯誤
  - 當初 AI 給的建議方向
- 復習結束後列出本次所有失誤及改進方向摘要

## 5. 使用者流程

```
啟動應用
    ↓
懸浮視窗顯示於畫面最上層
    ↓
點擊懸浮視窗
    ↓
展開輸入介面
    ├── 選擇 / 輸入情景
    ├── 輸入原文句子（中文）
    └── 輸入自己的英文翻譯
    ↓
送出 → AI 批改
    ↓
顯示回饋結果
    ├── 判斷（評分）
    ├── 問題解釋
    ├── 最佳版本
    ├── 不同情景版本
    └── 語法小提示
    ↓
結果自動儲存（供復習系統使用）
    ↓
縮回懸浮視窗
```

## 8. 開發優先順序

### 第一階段｜P0 MVP 核心

| 順序 | 模組 | 原因 |
|------|------|------|
| 1 | Config Store | 最先做，沒有 API Key 什麼都不能測試 |
| 2 | LLM Service | 核心引擎，其他模組都依賴它 |
| 3 | Context Manager | Main Panel 需要它才能運作 |
| 4 | Local Storage | 情景資料需要持久化 |
| 5 | Main Panel | 第一個使用者看得到的介面 |
| 6 | Result Panel | 有輸入就要有輸出 |
| 7 | Float Widget | MVP 的靈魂，最後整合進來 |
| 8 | Settings Panel | API Key 和情景管理的入口 |

### 第二階段｜P1（MVP 完成後）

| 順序 | 模組 | 原因 |
|------|------|------|
| 9 | Review Engine | 依賴 Local Storage 的完整資料才能運作 |
| 10 | Review Panel | Review Engine 完成後才有介面可接 |

## 9. UI 設計規範

### 懸浮按鈕（FAB）
- 圓形按鈕，52×52px，紫色背景（#534AB7）
- 內含翻譯 icon（白色）
- 一般狀態：只顯示按鈕，無文字
- Hover 狀態：按鈕變深色（#3C3489）+ tooltip 浮出顯示「人類翻譯機」
- 可自由拖曳移動，預設位置：右下角
- 始終置頂顯示於所有視窗之上

### 主視窗
- 點擊 FAB 展開，再次點擊或按 ✕ 縮回
- 寬度：300px
- 上半部：輸入介面（情景、原文、翻譯、送出按鈕）
- 下半部：批改結果區（預設隱藏，送出後才顯示）
- 主色調：紫色（#534AB7）

### 練習介面（Main Panel）

**進入方式**：點擊 FAB 展開，頂部 Tab 預設停在「練習」

**① 輸入中（結果區隱藏）**
- Tab 列：練習 / 復習
- 情景欄：下拉選單 + 「+ 新增」按鈕
- 原文欄：文字輸入框（中文）
- 翻譯欄：文字輸入框（英文）
- 送出批改按鈕：全寬，紫色填充
- 結果區：完全隱藏

**② 送出後（結果展開）**
- 輸入區保留（可修改後重新送出）
- 送出按鈕變成「重新批改」
- 分隔線下方展開批改結果
- 結果區塊順序：評分 → 問題解釋 → 更好的說法 → 不同情景版本 → 語法提示

### 復習介面（Review Panel）

**進入方式**：主視窗頂部 Tab 切換（練習 / 復習）

**① 考題畫面**
- 進度條 + 題數顯示（X / N 題）
- 原句卡片（顯示中文原文）
- 翻譯輸入框（讓使用者主動輸入英文）
- 送出批改按鈕

**② 批改結果**
- 與練習頁面一致的完整 AI 回饋格式（評分、問題解釋、更好的說法、不同情景版本、語法提示）
- 若分數高於上次，額外顯示進步提示
- 「下一題 →」按鈕繼續下一張

**③ 完成畫面**
- 顯示完成圖示 + 「今日復習完成！」
- 統計：復習題數 + 平均分
- 「展開詳細摘要」可列出每題分數
- 摘要用顏色區分程度：綠（8+）/ 橙（5-7）/ 紅（5 以下）
- 「返回練習」按鈕

---
- 預設：隱藏（送出後才展開）
- 評分：數字 / 10 + 一句話描述
- 區塊順序：問題解釋 → 更好的說法 → 不同情景版本 → 語法小提示
- 情景版本以 tag 標示（正式 / 朋友 / 書面）
- 語法提示以左側紫色 border 區塊顯示

## 11. 間隔重複演算法

### 採用方案：FSRS 簡化版

採用 **FSRS（Free Spaced Repetition Scheduler）** 的簡化版本，使用開源套件 `ts-fsrs`（TypeScript 實作），可直接整合進 Electron。

**選擇原因：**
- 不需要大量用戶資料即可運作，適合 MVP 單人使用
- 有成熟的開源 JS / TS 實作
- 比傳統 SM-2 更聰明——答錯不會把進度打回原點
- 用戶只需設定「目標記憶保留率」，系統自動計算最佳間隔

### 評分對應規則

批改分數由 AI 回饋產生（1–10 分），對應 FSRS 評級如下：

| AI 批改分數 | FSRS 評級 | 首次復習間隔 |
|------------|-----------|------------|
| 8–10 分 | Good / Easy | 3–7 天 |
| 5–7 分 | Hard | 1 天 |
| 1–4 分 | Again | 當天稍後再考 |

### 記憶三要素（FSRS 核心）

| 變數 | 名稱 | 說明 |
|------|------|------|
| R | Retrievability（可提取性） | 當下回想成功的機率 |
| S | Stability（穩定度） | R 從 100% 降到 90% 所需天數 |
| D | Difficulty（難度） | 該句子對使用者的主觀難度 |

每個句子都有獨立的 DSR 值，系統根據歷史作答紀錄動態調整。
